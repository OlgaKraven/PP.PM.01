1. Принципы интеграции фронтенда и бэкенда

Фронтенд — интерфейс в браузере (HTML, CSS, JS). Бэкенд — серверная часть, принимающая запросы, выполняющая логику и работающая с БД. Интеграция — через HTTP‑запросы, чаще всего REST API.

[Пользователь]
   ↓ (запрос)
[Frontend: JS/HTML] → (HTTP POST/GET)
   ↓
[Backend: Flask API / PHP / Node]
   ↓
[База данных]
   ↑ (ответ JSON)
  1. Пользователь заполняет форму регистрации.
  2. JavaScript отправляет POST /api/register.
  3. Сервер проверяет данные, пишет в БД и возвращает JSON.
  4. Фронтенд показывает сообщение об успехе.

2. REST API: архитектурный стиль взаимодействия

REST — принципы, где каждая сущность — ресурс по URL.

МетодНазначениеПример
GETПолучить данные/api/users
POSTСоздать запись/api/users
PUTОбновить запись/api/users/1
DELETEУдалить запись/api/users/1
 
from flask import Flask, request, jsonify
app = Flask(__name__)

users = []

@app.route('/api/users', methods=['POST'])
def create_user():
    data = request.json
    users.append(data)
    return jsonify({"message": "Пользователь создан"}), 201

@app.route('/api/users', methods=['GET'])
def list_users():
    return jsonify(users)
Особенности: JSON‑обмен; клиент не хранит состояние; каждый URL соответствует ресурсу (/api/orders/12).

3. Маршрутизация и работа с БД

Маршруты Flask

@app.route('/')
def home():
    return render_template('index.html')

@app.route('/orders/<int:id>')
def get_order(id):
    return f"Детали заказа №{id}"

ORM (SQLAlchemy)

from flask_sqlalchemy import SQLAlchemy
db = SQLAlchemy(app)

class User(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    name = db.Column(db.String(50))
Преимущества ORM: таблицы как объекты, генерация SQL, защита от SQL‑инъекций.

5. Оптимизация производительности

5.1 Кэширование

  • HTTP: Cache-Control, ETag.
  • Кэш приложения (Flask‑cache / self‑made).
  • Внешние: Redis, Memcached.
@app.route('/data')
@cache.cached(timeout=60)
def get_data():
    return get_heavy_data()

5.2 Минификация

CSS/JS минифицируем (Terser, UglifyJS, cssnano) → .min.css, .min.js.

5.3 Lazy‑loading

Загружать изображения по мере надобности:

<img src="photo.jpg" loading="lazy" alt="Фото">

6. SEO‑базис

  • Метаданные: <title>, <meta name="description">.
  • Один h1 на страницу; далее h2/h3.
  • ЧПУ‑URL: /catalog/phones вместо /page?id=12.
  • alt у изображений; скорость загрузки.
<title>Каталог товаров</title>
<meta name="description" content="Интернет-магазин с доставкой.">

8. Отладка и DevTools

  • Elements — правка HTML/CSS «на лету».
  • Console — ошибки/логи JS.
  • Network — время запросов и кэширование.
  • Responsive — проверка адаптивности.
  • Lighthouse — аудит производительности/SEO/A11y.
Во вкладке Network видны заголовки кэша, размеры ответов, время TTFB.
