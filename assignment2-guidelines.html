Этап 1. Анализ интерфейса и постановка задач

Задача: определить, какие страницы и элементы нужно оформить/улучшить.

  • Просмотрите макеты/файлы Модуля 1 — страницы: регистрация, авторизация, пользовательская (заявки), админ‑панель.
  • Выберите ключевые страницы для адаптации (минимум 3): index.html (вход), register.html (регистрация), dashboard.html (кабинет).
  • Определите улучшения: шрифты/цвета, поля форм и кнопки, адаптивное меню (бургер), контрасты и иконки.
Результат: список страниц и перечень улучшений дизайна/UX.

Этап 3. Проектирование адаптива под смартфон (390×844 px)

3.1 Базовая сетка

.container {
  display: flex;
  flex-direction: column;
  align-items: center;
  max-width: 390px;
  margin: 0 auto;
}

3.2 Медиа‑запросы

@media (max-width: 844px) {
  body { font-size: 16px; }
  .menu { flex-direction: column; }
}

3.3 Проверка в браузере

  • DevTools → Toggle Device Toolbar.
  • Устройство: iPhone 12 Pro (390×844).
  • Проверить все страницы и навигацию.
Результат: корректное отображение каждой страницы на смартфоне.

Этап 4. Улучшение визуального дизайна

Цветовая схема (пример: 3 тона)

  • Фон: #FFFFFF
  • Основной: #0057B7
  • Акцент: #FFD700

Типографика

body {
  font-family: 'Open Sans', sans-serif;
  color: #333;
  line-height: 1.5;
}

Иконки

Можно использовать Font Awesome или встроить SVG‑иконки локально.

<i class="fa fa-user"></i> Войти
<!-- или локальный SVG -->
<svg width="16" height="16" aria-hidden="true">...</svg> Войти

Фокус‑состояния

  outline: 2px solid #0057B7;
}

Уведомления (toast)

alert("Регистрация прошла успешно!");

Этап 5. Валидация UI (ошибки форм)

if (login.value === "") {
  login.classList.add("error");
  error.textContent = "Введите логин";
}
input.error {
  border: 2px solid red;
}
  • Проверяйте уникальность логина (API/массив).
  • Показывайте сообщения рядом с полями.
  • Не убирайте фокус у пользователя; сообщайте контекст ошибки.
Результат: пустые/неверные поля подсвечиваются, ошибки отображаются.

Этап 6. Реализация слайдера

6.1 HTML

<div class="slider">
  <div class="slides">
    <img src="img/slider1.webp" alt="">
    <img src="img/slider2.webp" alt="">
    <img src="img/slider3.webp" alt="">
    <img src="img/slider4.webp" alt="">
  </div>
  <button class="prev">‹</button>
  <button class="next">›</button>
</div>

6.2 CSS

.slider { position: relative; width: 100%; overflow: hidden; }
.slides { display: flex; transition: transform 0.5s ease; }
.slides img { width: 100%; }

6.3 JS

let index = 0;
const slides = document.querySelectorAll('.slides img');

function showSlide(i) {
  index = (i + slides.length) % slides.length;
  document.querySelector('.slides').style.transform = `translateX(${-index * 100}%)`;
}

document.querySelector('.next').onclick = () => showSlide(index + 1);
document.querySelector('.prev').onclick = () => showSlide(index - 1);

setInterval(() => showSlide(index + 1), 3000);
Результат: 4 слайда, автопрокрутка 3 сек, работают стрелки.

Этап 8. Проверка доступности и UX

  • Проверьте контраст (WebAIM Contrast Checker).
  • Добавьте alt для всех изображений.

Результат: базовое соответствие WCAG (контраст, alt, навигация).
